# 📚 章節排序和標題設置修正總結

## 🚨 原始問題

### 問題1: 章節排序錯亂
- **現象**: 章節在Notion中顯示順序不正確，第8章出現在第1章前面
- **原因**: 僅基於 `chapter_progress`（閱讀進度）排序，不反映章節實際順序
- **影響**: 跳躍式閱讀導致章節順序混亂，影響閱讀體驗

### 問題2: 章節標題錯誤
- **現象**: 普通高亮內容被誤認為章節標題
- **原因**: 標題提取驗證邏輯不夠嚴格，過長文本被接受
- **影響**: 顯示冗長無意義的"章節標題"

### 問題3: "未知章節"過多
- **現象**: 大量高亮歸類為"未知章節"
- **原因**: 章節提取策略單一，未充分利用ContentID信息
- **影響**: 章節組織不清晰

## ✅ 解決方案

### 1. 智能章節排序算法

#### 多重排序標準
```python
# 排序優先級：
1. 章節順序號 (從ContentID提取)
2. 最小閱讀進度 (最早遇到的高亮)
3. 章節名稱 (字典序)
```

#### ContentID順序提取
```python
# 支持多種章節編號格式：
- Section0001, Section0012 → 1, 12
- 01.xhtml, 12.xhtml → 1, 12
- chapter01, chapter12 → 1, 12
- prologue → 0 (序言排最前)
- epilogue → 9999 (結語排最後)
```

#### 特殊章節處理
```python
special_orders = {
    'prologue': 0,      # 序言
    'preface': 1,       # 前言
    'introduction': 2,  # 引言
    'epilogue': 9999,   # 尾聲
    'appendix': 10000,  # 附錄
}
```

### 2. 增強章節標題驗證

#### 嚴格的長度限制
```python
# 原限制: 200字符 → 新限制: 100字符
if len(text.strip()) > 100:
    return None
```

#### 標點符號密度檢查
```python
# 拒絕標點符號過多的文本 (>30%)
punctuation_ratio = sum(1 for c in text if c in '。！？；：，、') / len(text)
if punctuation_ratio > 0.3:
    return None
```

#### 章節特徵檢測
```python
# 必須包含明確的章節特徵：
- 第X章、Chapter X
- 序言、前言、結語等關鍵詞
- 數字開頭格式 (1. 2. 等)
- 包含冒號且較短的文本
```

#### 完整句子過濾
```python
# 章節標題通常不是完整句子
if text.endswith(('。', '！', '？', '.')) and len(text) > 30:
    return None
```

### 3. 智能標題修正機制

#### 錯誤標題檢測
- 章節名稱與高亮文本完全相同且過長
- 明顯的普通高亮內容特徵

#### 自動修正策略
```python
# 1. 從ContentID重新提取章節名稱
fallback_chapter = extract_chapter_name(content_id)

# 2. 替換"未知章節"為更有意義的名稱
if chapter_name == "未知章節":
    display_chapter_name = "其他內容"

# 3. 限制顯示長度
if len(display_chapter_name) > 50:
    display_chapter_name = display_chapter_name[:47] + "..."
```
